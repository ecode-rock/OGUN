## README SUMMARY

**Project goal:** Automated daily pipeline that pulls MLB game data from Baseball Savant (Statcast), cleans it, and outputs CSV files to power a website. One scheduled morning run captures all completed games from the previous day.

**Key discovery (Phase 1, already completed):**
No browser scraping is needed. All data is available through two clean JSON API endpoints:

  Endpoint 1 — Daily schedule:
    https://baseballsavant.mlb.com/schedule?date=YYYY-M-D
    Returns all games for a date: gamePks, team names, status, doubleheader flags.

  Endpoint 2 — Full game data:
    https://baseballsavant.mlb.com/gf?game_pk=GAMEPK
    Returns all pitch data, scoreboard/linescore, boxscore, and win probability.

**Target output — four CSV types per day:**
  - pitches_YYYYMMDD.csv     (one row per pitch)
  - games_YYYYMMDD.csv       (one row per game)
  - innings_YYYYMMDD.csv     (one row per half-inning)
  - wpa_YYYYMMDD.csv         (one row per at-bat, win probability)

**Tech stack:** Python 3, requests, pandas. No authentication required on endpoints (as of 8/20/2025).

**Known data quality issues flagged in README:**
  - Player biographical bloat (birthDate, birthCity, height, weight, gender, draftYear, etc.)
  - 8+ redundant name fields per player (only fullName needed)
  - Live game-state cursor fields (onDeck, inHole — meaningless post-game)
  - API link fields (/api/v1/... hrefs throughout)
  - Operational metadata (calendarEventID, reverseHomeAwayStatus, ifNecessaryDescription, etc.)
  - Floating point noise (e.g. 0.700000000000003)

**Edge cases to handle:**
  - Doubleheader Game 1 / Game 2 — tag in CSV
  - Postponed / Cancelled games — log to edge_cases.log, skip
  - No games today — clean exit
  - Missing fields — always use .get() with None default

**Known good test gamePk:** 776661 (Blue Jays @ Pirates, 8/20/2025)

## /gf ENDPOINT — STRUCTURE DISCOVERED THIS SESSION

URL: https://baseballsavant.mlb.com/gf?game_pk=776661

Top-level keys in response:
  game_status_code, game_status, gamedayType, gameDate, hasAbs,
  scoreboard, venue_id, home_team_data, away_team_data,
  team_home_id, team_away_id,
  team_home  [list — 117 pitch rows for home team pitching],
  team_away  [list — 155 pitch rows for away team pitching],
  exit_velocity  [list — 39 rows, batted balls only],
  home_pitchers, away_pitchers  [dicts keyed by player ID],
  home_batters, away_batters    [dicts keyed by player ID],
  home_lineup, away_lineup, home_pitcher_lineup, away_pitcher_lineup,
  boxscore, players, cacheKey, cache_hit, game_date

Key architectural note:
  - Pitch data is split into team_home and team_away arrays (pitches FROM each team's perspective)
  - Exit velocity (batted ball) data is in a SEPARATE exit_velocity array
  - The two must be merged by play_id to get complete pitch + batted ball data on one row
  - Pitches without a batted ball event will have blank EV columns — this is correct/expected

## scrape_3.csv — COLUMN INVENTORY (109 columns, 272 rows)

=== GAME METADATA (4) — added by pipeline script ===
  game_pk, game_date, home_team, away_team

=== IDENTITY / PLAY CONTEXT (14) ===
  type          — always "pitch" (noise — discard)
  source        — always "gamefeed" (noise — discard)
  play_id       — UUID per pitch
  rowId         — "pitchcount-gamePk" string (e.g. "23-776661")
  cap_index     — internal sequencing index
  index         — internal
  inning        — inning number
  ab_number     — at-bat number within game
  pitch_number  — pitch number within at-bat
  game_total_pitches — running pitch count for game
  is_last_pitch — boolean, final pitch of at-bat
  is_abs_challenge — ABS (automated ball-strike) challenge flag
  player_name   — actually the FIELDING team abbreviation (misleading name, likely noise)
  game_pk       — native field, redundant with added game_pk column

=== BATTER / PITCHER (9) ===
  batter            — numeric player ID
  batter_name       — full name string
  stand             — batter handedness (L/R)
  pitcher           — numeric player ID
  pitcher_name      — full name string
  p_throws          — pitcher handedness (L/R)
  team_batting      — batting team abbreviation
  team_fielding     — fielding team abbreviation
  team_batting_id   — numeric team ID
  team_fielding_id  — numeric team ID

=== COUNT / RESULT (15) ===
  balls, strikes, outs         — count AT TIME OF PITCH
  pre_balls, pre_strikes       — count BEFORE pitch (use these for "count going in")
  balls_and_strikes            — string like "00", "12", "32"
  result                       — plate appearance result (e.g. "Strikeout", "Single")
  events                       — same as result on last pitch of PA, blank otherwise
  des                          — play description string
  call                         — single-letter call code (S, B, X, F, etc.)
  call_name                    — human-readable call (Strike, Ball, In play, Foul, etc.)
  description                  — more detailed description
  result_code                  — single-letter result code
  pitch_call                   — lowercase call (foul, ball, called_strike, etc.)
  is_strike_swinging            — boolean
  is_bip_out                   — "Y"/"N" string — ball in play that resulted in out
  contextMetrics               — always empty dict {} (noise — discard)

=== PITCH TYPE / VELOCITY (6) ===
  pitch_type    — abbreviation (SI, FF, SL, CH, etc.)
  pitch_name    — full name (Sinker, Four-Seam Fastball, Slider, Changeup, etc.)
  start_speed   — velocity at release (mph)
  end_speed     — velocity at plate (mph)
  spin_rate     — RPM
  extension     — release extension in feet

=== PLATE LOCATION (9) ===
  plate_x / px   — horizontal plate location (px = raw, plate_x = same value — one is redundant)
  plate_z / pz   — vertical plate location (same situation)
  sz_top         — top of strike zone for this batter
  sz_bot         — bottom of strike zone for this batter
  sz_depth       — strike zone depth
  sz_width       — strike zone width
  zone           — zone number (1-9 in zone, 11-14 outside)
  savantIsInZone — boolean
  isInZone       — boolean (appears identical to savantIsInZone — one is redundant)

=== MOVEMENT / BREAK (9) — several redundant ===
  breakX              — horizontal break (inches)
  inducedBreakZ       — induced vertical break (inches, gravity removed)
  breakZ              — total vertical break (includes gravity)
  pfxX                — pfx horizontal movement
  pfxZ                — pfx vertical movement
  pfxZWithGravity     — pfxZ including gravity effect
  pfxXWithGravity     — pfxX including gravity effect (redundant with pfxX)
  pfxXNoAbs           — pfxX without absolute value (redundant)
  inducedBreakZForcedSign — string like "+6.2" (redundant with inducedBreakZ — discard)

=== DIRECTION FLAGS — NOISE (3) ===
  pfxZDirection       — Unicode arrow character (↑↓) — visual label, not data
  pfxXDirection       — Unicode arrow character (→←) — visual label, not data
  ivbZDirection       — Unicode arrow character — visual label, not data

=== RELEASE POINT (3) ===
  x0    — horizontal release position
  z0    — vertical release position
  y0    — distance from plate at release (always ~50 ft — less useful)

=== PHYSICS VECTORS — ADVANCED/SPECIALIZED (6) ===
  ax, ay, az      — acceleration components
  vx0, vy0, vz0   — velocity components at release

=== DERIVED PLATE POLYNOMIALS — LIKELY NOISE (5) ===
  plateXPoly        — same value as plate_x (redundant)
  plateYPoly        — interpolated y at plate
  plateZPoly        — same value as plate_z (redundant)
  plateTime         — time from release to plate (seconds)
  plateTimeSZDepth  — plate time adjusted for SZ depth

=== BATTED BALL / EXIT VELOCITY (12) — merged from exit_velocity array ===
  hit_speed        — exit velocity (mph) — appears same as launch_speed
  hit_speed_round  — rounded version of hit_speed
  launch_speed     — exit velocity (mph) — same as hit_speed, one is redundant
  hit_angle        — launch angle (degrees) — appears same as launch_angle
  launch_angle     — launch angle (degrees) — same as hit_angle, one is redundant
  hit_distance     — projected distance (feet)
  xba              — expected batting average
  is_barrel        — barrel flag (0/1)
  hc_x             — hit coordinate x (spray chart)
  hc_y             — hit coordinate y (spray chart)
  hc_x_ft          — hc_x in feet (different coordinate system)
  hc_y_ft          — hc_y in feet (different coordinate system)
  NOTE: blank on non-batted-ball pitches — this is correct behavior

=== HTML NOISE (1) ===
  result_table    — raw <div> HTML string — discard

=== BASERUNNER STATE (3) — not in current README schema ===
  runnerOn1B, runnerOn2B, runnerOn3B   — boolean flags
  NOTE: only present on some rows — DECISION NEEDED: include in pitch schema?

=== BAT SPEED (1) — not in current README schema ===
  batSpeed        — bat speed in mph (notable Statcast metric added ~2024)
  NOTE: DECISION NEEDED: include in pitch schema?

=== PITCH USAGE COUNTERS (4) ===
  player_total_pitches            — pitcher's total pitches so far in game
  player_total_pitches_pitch_types — pitch type breakdown (may be nested object)
  pitcher_pa_number               — which PA number for this pitcher in game
  pitcher_time_thru_order         — which time through the order
